# 媒体文件缓存管理方案

实现生成完成的图片/视频自动缓存到本地，按日期目录组织，并支持超时/容量限制自动清理。

---

## 需求分析

**问题**：第三方平台返回的视频/图片URL有有效期，超期后无法访问。

**目标**：
1. 任务完成后自动下载媒体文件到本地 `upload/cache/` 目录
2. 按日期目录组织（如 `upload/cache/2026-02-27/`）
3. 支持配置：超时天数 或 总容量限制
4. 超限后自动删除最早的文件

---

## 方案设计

### 1. 目录结构

```
upload/
└── cache/
    └── 2026-02-27/
        ├── video_abc123.mp4
        └── image_def456.png
```

### 2. 配置项 (`config.example.yml`)

```yaml
media_cache:
  enabled: true                    # 是否启用缓存
  cache_dir: "upload/cache"        # 缓存目录（相对于项目根目录）
  max_days: 30                     # 文件保留天数（0=不限制）
  max_size_gb: 10                  # 最大缓存容量GB（0=不限制）
  cleanup_on_startup: true         # 启动时执行清理
  cleanup_interval_hours: 24       # 定时清理间隔（小时）
```

### 3. 核心模块设计

#### 3.1 新增文件：`utils/media_cache.py`

| 函数 | 功能 |
|------|------|

#### 3.2 修改文件：`task/visual_task.py`

在 `_handle_task_success()` 中增加缓存逻辑：

```python
def _handle_task_success(project_id, task_id, media_url):
    # 1. 下载并缓存媒体文件
    from utils.media_cache import download_and_cache
    local_url = await download_and_cache(media_url, task_id, "video")
    
    # 2. 更新数据库（可选：同时保存原URL和本地URL）
    AIToolsModel.update_by_project_id(
        project_id=project_id,
        result_url=media_url,        # 原始URL
        cached_url=local_url,        # 本地缓存URL（需新增字段）
        status=AI_TOOL_STATUS_COMPLETED
    )
```

## 技术要点

### 异步下载
使用 `aiohttp` 进行非阻塞下载，符合项目规范（禁止在异步函数中调用同步库）。

### 跨平台兼容
使用 `os.path.join()` 和 `pathlib` 处理路径，兼容 Windows/Linux/macOS。

### 文件命名
格式：`{task_id}_{timestamp}_{hash8}.{ext}`  
示例：`12345_20260227_abc12def.mp4`

### 清理策略
- **按天数**：删除 `mtime` 超过 N 天的文件
- **按容量**：当总大小超限时，按 `mtime` 从旧到新删除，直到低于阈值

## 确认的设计决策

1. **替换 result_url**：直接将 result_url 替换为本地缓存URL，无需新增数据库字段
2. **清理策略：并集**：超过天数 或 超过容量 任一满足即删除
